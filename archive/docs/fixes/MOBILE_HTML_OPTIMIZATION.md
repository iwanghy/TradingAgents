# HTML手机适配优化说明

## 修改概述

已将HTML报告生成优化为**手机固定布局（375px宽度）**，专门适配手机阅读。

## 关键修改

### 1. 文件：`tradingagents/graph/report_generator.py`

#### 修改内容：

**A. HTML生成Prompt优化**（`_build_html_prompt`方法）
- 明确指定为"手机固定布局"而非响应式设计
- 固定页面宽度：375px（iPhone标准宽度）
- 内容区域：343px（375px - 左右各16px边距）

**B. 字体大小规范**（适合手机阅读）：
- 正文：18px（最小可读尺寸）
- H1标题：32px
- H2标题：26px
- H3标题：22px
- 小字/注释：14px

**C. 间距规范**（提高可读性）：
- 正文行高：32px（1.78倍）
- 段落间距：24px
- 标题上下边距：20px
- 列表项间距：16px

**D. Viewport设置**：
```html
<meta name="viewport" content="width=375, initial-scale=1.0">
```

**E. 后备模板优化**（`_generate_fallback_html`方法）
- 后备模板也已更新为手机适配布局
- 确保即使LLM生成失败，也能得到手机友好的HTML

### 2. 测试文件：`test_mobile_html.py`

创建了专门的测试脚本，验证手机适配HTML生成功能。

## 验证结果

所有测试通过（5/5）：
- ✅ 包含viewport设置（width=375）
- ✅ body宽度375px
- ✅ 正文字体18px+
- ✅ 包含charset
- ✅ HTML结构完整

## 生成的HTML特点

1. **固定宽度布局**
   - 页面宽度固定为375px
   - 不会响应式缩放
   - 专为手机屏幕优化

2. **大字体设计**
   - 正文字体18px，易于阅读
   - 行高32px，提高可读性
   - 标题层次分明

3. **深色主题**
   - 背景色：#000000
   - 主要文字：#E9ECF1
   - 次要文字：#A9B3C1

4. **决策卡片**
   - 买入：#4AF6C3（青绿色）
   - 卖出：#FF433D（红色）
   - 持有：#0068FF（蓝色）

## 使用方法

生成的HTML文件可以直接在手机浏览器中打开，或在桌面浏览器中使用开发者工具设置为iPhone视图（375px宽度）查看效果。

## 测试

运行测试脚本：
```bash
python test_mobile_html.py
```

测试结果文件：
```
reports/test_mobile_375px.html
```

## 技术细节

- 不使用响应式设计
- 所有尺寸使用固定像素值
- 不使用媒体查询
- 专为iPhone标准宽度（375px）设计
- 支持Android设备（大多数也是375px-428px宽度）

## 兼容性

- iPhone（所有型号）：完美适配
- Android（375px-428px宽度）：良好适配
- iPad：可能有较大边距，但可读
- 桌面浏览器：显示为375px宽度的移动视图

## 后续优化建议

如果需要支持更多设备尺寸，可以考虑：
1. 提供多个固定宽度版本（375px, 414px, 428px）
2. 添加设备检测，自动选择合适的宽度
3. 或转为真正的响应式设计（如果需求变化）
